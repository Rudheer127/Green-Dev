WHAT IS GREENDEV COACH?
─────────────────────────

An AI-powered sustainability analysis tool for student developers.

User inputs: GitHub repository URL + cloud deployment configuration
System outputs: Sustainability score (0-100) + prioritized recommendations 
              + AI-generated reports in 4 formats

Time to insight: <30 seconds
Cost: Free (MVP), future: freemium model

TARGET AUDIENCE: Student developers, hackathon teams, university students

WHY IT MATTERS
─────────────

Problem: Student developers waste cloud resources without realizing it
Example: An always-on EC2 instance wastes 14× more compute vs serverless
Impact: Over 10,000 tons CO2 annually at scale + $2M+ savings in user costs

Insight: Green engineering is also good engineering (no trade-off)
Example: Smaller Docker images = faster builds + lower costs + better UX

CORE TECH STACK
───────────────

Frontend: Next.js 16 (Vercel deployment)
Backend: Supabase PostgreSQL + pgvector caching
AI: AWS Bedrock (Claude Sonnet 4.6)
Analysis engines: 5 modular Python engines for different sustainability dimensions
GitHub integration: Public repo analysis (no credentials needed)

HOW IT WORKS (3-Step Flow)
──────────────────────────

Step 1: INTAKE
├─ User enters GitHub URL and deployment config (cloud provider, region, service)
├─ Validation: URL format check, rate limiting
└─ Time: <5 seconds

Step 2: ANALYSIS
├─ Five parallel engines run: CI/CD, Docker, Compute, Assets, Region
├─ Deterministic rules (not pure LLM) produce structured findings JSON
├─ pgvector cache checked for similar patterns
└─ Time: 5-10 seconds

Step 3: SYNTHESIS & DELIVERY
├─ If cache hit: Return cached reports
├─ If cache miss: Invoke Claude for 4 report variants
├─ Results streamed to frontend in real-time
├─ User sees sustainability score + 3-5 recommendations
└─ Time: 10-15 seconds

TRACTION & SUCCESS
──────────────────

Hackathon phase (48 hours):
├─ 50+ repositories analyzed
├─ Multiple teams using for pitch preparation
├─ Strong judge feedback
├─ University interest signaled

Long-term metrics (projected):
├─ 10k+ users by Year 1
├─ 100k+ total scans
├─ 10k+ tons CO2 avoided annually

================================================================================
SUCCESS INDICATORS
================================================================================

If your pitch is working, you'll hear / see:

✓ Specific questions about pricing/timeline (not "what is it?")
✓ Audience leans forward during data points (not on phones)
✓ Follow-up meetings scheduled immediately after
✓ People ask "can I try it?" and "when do signups open?"
✓ Competitive comparison questions ("how is this different from X?")
✓ Partnership inquiries (universities, enterprises)
✓ Media/press interest ("can we feature this?")
✓ Technical depth questions (good sign: they're seriously evaluating)

If your pitch needs work:

✗ "Tell me more about GitHub" (unclear differentiation)
✗ "How is sustainability measured?" (lack of credibility)
✗ "What's your revenue model?" (unclear business justification)
✗ "Who exactly would pay?" (target audience confusion)
✗ "Why now?" (timing argument weak)
